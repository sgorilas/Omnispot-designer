Layout

A layout represents a particular arrangement of components on the screen. It observes the notion of duration and completion. It also is associated with a cron expression for scheduling purposes. A layout is termed to have completed either when its duration has expired, or any one of its components has completed. When a layout completes, the next layout in the list of layouts in its deployment will start. The only exception to this rule is that if the cron expression of a layout fires, then that layout will start immediately, irrespective of whether the currently running layout has completed or not.

A layout has a set of properties that can be modified in the property sheet view. These are:

  • Cron Expression: Specifies the cron expression that is associated with the layout. If empty, then there is no trigger associated with the layout, and it can only run if its turn comes in the normal scheme of things.
  • Duration: The duration (in seconds) of the layout. If the layout runs longer than this amount of seconds, it is deemed to have completed.
  • Name: A name for the layout. This is only used in the designer and in error reporting.

A layout also contains a list of regions. The ordering of the list specifies the z-ordering of the regions in case of overlaps (the last region in this list overlaps any preceding regions that happen to have common areas.
